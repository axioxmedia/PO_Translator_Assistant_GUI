# PO 翻译工具 v1.3

专为 `.po` 文件本地化设计的稳定 Tkinter GUI 工具。

## 为什么需要这个工具？

传统 PO 翻译脚本大多按顺序或索引应用翻译。当 `.po` 文件重新生成、存在重复字符串或结构微调时，极易出现**翻译错位**（翻译内容跑到错误的 msgid 下）。

**v1.2 核心解决**：应用翻译时完全以 `msgid` 作为匹配键，彻底杜绝错位问题。

## 主要功能

- 简洁中文界面，专为快速工作流优化
- 一键提取未翻译条目到 JSON（方便 AI 批量翻译）
- 通过 `msgid` 精准匹配将 JSON 翻译安全写回 `.po` 文件
- 实时进度条 + 详细日志
- 自动更新 PO 文件头（Language、修订日期）
- 应用过程多线程处理，界面不卡顿

## 运行环境

- Python 3.6+
- 无需安装任何第三方库（tkinter 为 Python 标准库）

## 快速开始

```bash
python po_translator_gui.py
```

## 推荐使用流程

### ① 提取未翻译内容
1. 选择原始 `.po` 文件
2. 点击 **① 提取未翻译内容**
3. 保存生成的 JSON 文件

仅会导出 `msgstr` 为空或与 `msgid` 相同的条目。

### ② 使用 AI 翻译 JSON
将导出的 JSON 粘贴给 ChatGPT、Claude、Grok 等大模型，推荐提示词：

> 请将 JSON 数组中所有 `"msgstr"` 的值翻译成简体中文，保持 JSON 结构和键名完全不变，只翻译文本内容。

### ③ 应用翻译（推荐使用）
1. 选择**原始** `.po` 文件
2. 选择翻译好的 JSON 文件
3. 选择输出路径
4. 点击 **② 应用翻译（推荐）**

工具会：
- 严格按 `msgid` 进行匹配
- 仅安全替换 `msgstr` 部分
- 自动更新文件头信息
- 显示实时进度和详细日志

### ④ 验证结果
点击 **查看当前进度** 或用 Poedit / 文本编辑器打开输出文件检查。

## JSON 翻译文件格式

必须是 JSON **数组**：

```json
[
  {
    "msgid": "Welcome to our platform",
    "msgstr": "欢迎来到我们的平台"
  }
]
```

没有 `"msgstr"` 或值为空的条目会被跳过。
